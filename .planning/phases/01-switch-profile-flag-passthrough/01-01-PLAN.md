**Critical research finding (D-03 override):** CONTEXT.md D-03 specifies "blind passthrough" but research VERIFIED that `activate` only accepts `--driver-version N`. Flags `-b`, `-n`, `--show-trace` must be translated to environment variables (`HOME_MANAGER_BACKUP_EXT`, `DRY_RUN`, `VERBOSE`). This is how the `home-manager` CLI itself invokes activate. Using D-03 as written would cause `activate` to print `unknown option` and exit 1.

**Also adding `--driver-version 1`** to the `exec` call per research pitfall 5 — current script omits it, but `home-manager` always passes it. Without it, activate uses legacy driver version 0 behavior.
</context>

3. **Flag parsing loop** — `while (( $# > 1 )); do case "$1" in ...` processing all args except the last:
   - `-b)` → `shift`; guard `(( $# < 2 ))` for missing value (print `"switch-profile: -b requires an argument"` to stderr, exit 1); `export HOME_MANAGER_BACKUP_EXT="$1"` (per ERGO-01)

   - `--show-trace)` → `export VERBOSE=1` (per ERGO-03; matches home-manager's coupling of --show-trace with VERBOSE)
   - `*)` → print `"switch-profile: unknown flag '$1'"` to stderr, exit 1 (reject unknowns early with a clear message rather than letting activate produce a cryptic error)
   - Each case ends with `shift` (`;;&` not needed — explicit `shift` at end of loop body after `esac`)

4. **Profile name** — `PROFILE_NAME="$1"` (last remaining arg, per D-01).

5. **Profile resolution** — unchanged: `DATA_DIR`, `RESULT_LINK` construction, existence check with the existing two-line error message.

6. **Activation** — `echo "Activating profile: $PROFILE_NAME"` then `exec "$RESULT_LINK/activate" --driver-version 1` (add `--driver-version 1` per research pitfall 5; preserve `exec` per Claude's discretion).

Follow all shell conventions from CONVENTIONS.md: error messages as `"switch-profile: <message>" >&2`, exit code 1 for all errors, no `getopts`.
